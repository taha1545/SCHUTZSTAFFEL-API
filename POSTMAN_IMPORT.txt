SCHUTZSTAFFEL API - POSTMAN IMPORT FORMAT
================================================

BASE URL: http://localhost:3000/api

AUTHENTICATION ENDPOINTS
========================

POST /auth/student/signup
- No Auth Required
- Body: {fullName, email, grade (optional)}
- Purpose: Register new student account

POST /auth/student/login
- No Auth Required
- Body: {uniqueCode}
- Purpose: Student login, returns JWT token

POST /auth/teacher/signup
- No Auth Required
- Body: {name, email, password}
- Purpose: Register new teacher account

POST /auth/teacher/login
- No Auth Required
- Body: {email, password}
- Purpose: Teacher login, returns JWT token 

GET /google
- No Auth Required
- Purpose: Initiate Google OAuth 2.0 login

GET /google/callback
- No Auth Required
- Purpose: Google OAuth callback endpoint


USER MANAGEMENT ENDPOINTS
=========================

GET /users/me
- Auth Required: Yes (JWT Token)
- Purpose: Get current logged-in user profile

GET /users
- Auth Required: Yes
- Role Required: Admin
- Purpose: Get all users list

GET /users/:id
- Auth Required: Yes
- Role Required: Admin
- Path Param: id (user ID)
- Purpose: Get specific user by ID

PUT /users/me
- Auth Required: Yes
- Body: {fullName (optional), email (optional), grade (optional)}

DELETE /users/:id
- Auth Required: Yes
- Role Required: Admin
- Path Param: id (user ID)
- Purpose: Delete user

GET /users/search
- Auth Required: No

- Purpose: Search users by name or email

GET /users/by-goal/:goalId
- Auth Required: No
- Path Param: goalId (goal ID)
- Purpose: Get all users associated with a goal

AUTHORIZATION LEVELS
====================

No Auth: Public endpoints - no authentication required
JWT Required: Must have valid JWT token in Authorization header
Admin Only: User must have admin role
Teacher Only: User must have teacher role
Student Only: User must have student role
Admin + Teacher: User must be either admin or verified teacher


HEADER FORMAT FOR AUTHENTICATED REQUESTS
========================================

Authorization: Bearer <JWT_TOKEN>
Content-Type: application/json

Example:
Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
Content-Type: application/json


RESPONSE FORMAT
===============

Success (2xx):
{
  "success": true,
  "message": "Operation successful",
  "data": { ... }
}

Error (4xx/5xx):
{
  "success": false,
  "message": "Error description",
  "errors": [
    {
      "field": "email",
      "message": "Invalid email format"
    }
  ]
}
